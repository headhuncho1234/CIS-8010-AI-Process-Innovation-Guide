Class 1 — Problem-to-be-Solved, Business Processes & BPMN

1.1 Problem-to-be-Solved (PTBS) Approach

PTBS — Problem To-Be-Solved (verbatim): "The elemental / core reason the service is valuable to the consumer of the service. It is often not an obvious or surface-level reason."
Repeated "Who are your customers? You don't get paid for what you do. You get paid for the value you create." (Repeated in Class 1 and again in Class 5.)
  • Core questions: What problem do you solve? For whom do you solve the problem? Where do you fit in the value network? Do you add value? Can you capture some of that value as revenue?
  • Rule: "A business must capture part of the value it creates."
  • Example: Boeing/Airbus — core PTBS is the need for human connection, not flying. Demand exceeds capacity because the need for connection is insatiable.

1.2 Business Processes

Business (def): A business is a collection of processes and the resources that make those processes possible.
Repeated "Improving processes will improve a business. Transforming processes will transform a business."
"Before you can change a process, you must understand how the current process works and what the intended process should look like."

1.3 BPMN — Why It Exists

"A given BPMN diagram should have one and only one interpretation. The process logic should be completely and unambiguously described by the diagram alone." — Bruce Silver (2011)

BPMN = Business Process Model & Notation. Purpose: a single shared visual language so understanding can be communicated, not just held.

1.4 Pools & Swim Lanes

  • Pool: Represents a participant in a business process (organization, department, role). Container for the process flow. Each pool = separate entity / organizational boundary.
  • Each pool = one independently-owned, separately-managed process. The main pool is the "implied domain of control" (called an orchestration).
  • Swim Lanes: Inside a pool. Show who is performing each task. Label swim lanes using roles, NOT people's names.
Trap A BPMN diagram can contain multiple pools — but a single process cannot "jump" across pools. If a process is managed as one process, it must be in one pool divided into swim lanes.

1.5 Three Basic BPMN Shapes

ShapeSymbolDefinition (verbatim)
ActivityRounded rectangleA step in the process. Represents work/action performed. Smallest unit of work is called a "task".
EventCircleA signal (message) that something has happened. Can start, pause, resume, interrupt or redirect a process or activity.
GatewayDiamondRepresents a decision. Controls flow branching, merging, and parallel actions. Pure logic; result based on data previously obtained/analyzed.

All ends of a sequence flow must connect to one of these 3 flow nodes.

1.6 Activities: Tasks vs. Sub-processes

Task: Atomic activity (cannot be broken down into other activities).
Sub-process: Activity that can be broken down into other tasks.

7 BPMN Task Types MEMORIZE

Task TypeDefinition
Script TaskAutomated task executed by a business process engine using a script.
Send TaskTask that sends a message to an external participant.
Receive TaskTask that waits for a message to arrive from an external participant.
User TaskTask performed by a human user with the assistance of a software application.
Manual TaskTask performed by a human without the aid of any business process execution engine or application.
Service TaskAutomated task that uses a web service or automated application.
Business Rule TaskTask that provides input to a business rules engine and receives output from it.

Sub-process Types

  • Reusable Sub-process (Call Activity): References a globally-defined process reusable across multiple models.
  • Embedded Sub-process: Defined within the parent process; can be collapsed or expanded.
  • Event Sub-process: Triggered by an event, used for exception handling.

Activity Labeling Rules

Trap
  • Label activities using IMPERATIVE VERBS — phrased like instructions ("Send Invoice", "Approve Request").
  • Labels describe the ACTION performed — they are NOT general descriptions of what is happening or annotations about status.
  • Do NOT include "who" performs the task in the label. The swim lane tells you who.
  • Avoid verbs like "Start", "Attempt", etc. (Listed on "Common Mistakes to Avoid" slide.)

Throwing vs. Catching

Trap Sending (Throwing) symbol is FILLED IN / DARKENED.
Receiving (Catching) symbol is NOT filled.
(This is counterintuitive — many students reverse it.)

Loops

Standard LoopMulti-Instance Loop
Activity repeated until condition becomes true. One instance at a time. Like a while-loop. Multiple instances of the same activity — parallel or sequential. Used when processing entire collections; activity runs once per object in collection.
Example: "Keep screening candidates until find qualified candidate." Example: "Check delivery item" for each delivery item in shipment.

1.7 Events

"Tasks vs. Events: You can assign tasks to specific people (roles) or to machines / computers. Events just happen."
Rule Every process MUST start with a START event. Every process MUST end with an END event. Intermediate events happen between.

Event Borders

  • Start event: single thin border line
  • Intermediate event: double border line
  • End event: bold/thick border line

Intermediate Events

Modeled only when:

  1. An event relevant to others is triggered within a process (receiving message, timer counting down, etc.), OR
  2. There is a reaction to an event in the process (a point in time is reached, etc.).

Intermediate events can be catching OR throwing.

Interrupting vs. Non-Interrupting Intermediate Events

Interrupting Intermediate Event: Triggers the process to move along the exception flow instead of the regular flow.
Non-interrupting Intermediate Event: Process continues as normal AND an exception flow is triggered as an additional process flow.

Boundary Events

Put on the boundary of a task symbol. Alternative process path is triggered if the event occurs before the task is completed.

  • Non-interrupting boundary event: Keep doing current process AND ALSO follow exception flow.
  • Interrupting boundary event: Stop current process and ONLY follow exception flow.

1.8 Gateways

"Gateways split or fork the sequential flow between activities into separate paths and bring flows back together (merge or join). Gateways do not make decisions or allow for time to pass. They simply direct traffic."
Trap Gateways CANNOT make decisions and CANNOT perform actions. The preceding activity (or data) determined the answer; the gateway just routes. Listed on the "Common Mistakes" slide as "Gateways performing actions."

Labeling Rules

  • Label gateways as questions.
  • Label exiting paths as answers to the question.
  • No need to label parallel gateways because you must follow all paths (logic is already clear).

The 4 Gateway Types MEMORIZE

GatewaySymbol/NameRule (verbatim)
ExclusiveXOR (×)Follow only 1 path.
InclusiveOR (○)Follow 1 or more paths.
ParallelAND (+)Follow ALL paths. You must finish all paths before continuing.
Event-basedpentagon in circleFollow path triggered by event.

Event-Based Gateway TRAP

Trap
  • Events are in the future! It's the only "future-looking" gateway.
  • Every path out of this gateway has an event on it.
  • The intermediate event is the "condition" that must be met to proceed on that path.
  • Gateway logic is based on which event happens first.
  • Exiting paths are re-combined with an XOR gateway.
  • Put a timer event in case none of the events are satisfied — so you don't get stuck waiting forever.

OR Gateway Special Rules

  • Each path is evaluated for meeting the gateway condition; multiple paths can be true and taken concurrently (unlike XOR where only 1 path can be taken).
  • Exiting paths must be recombined with a merge OR gate.
  • Default flow is enabled if and only if no other path is enabled — denoted by a hash mark.

AND (Parallel) Gateway Rules

  • Creates required concurrent paths (each executes independently).
  • Creates a "token" for each split path.
  • All tokens must be joined (merged) before ending the path — unless a termination is encountered.
  • Flow out of a joining AND gate doesn't occur until ALL preceding paths are complete.
  • Style note: Do NOT use an AND gateway to join parallel paths into a none end event — a join is always implied at a none end event.

Complex (Discriminator) Gateways

Combinations of OR, XOR, and EVENT gateway conditions. Exiting paths re-combined using a complex gateway. "You probably won't use this in this class."

1.9 Process Flow Rules

  • Sequence flow arrows (solid) show process sequence.
  • Message flow arrows (dashed with small circle on sending end) go between pools only.
  • Every object (except Start/End events) must have process arrow inflow AND outflow.
  • Message flow arrows do NOT substitute for sequence flow arrows.
Orchestration: The managed flow of control within a business process. Represented by sequence (control) flows — solid lines with arrows.
Choreography: Pattern of messages between separate, distinct processes (pools). Coordination of processes — done by message flows (dashed lines with arrows at one end and small circles at the other end).

1.10 Data Flow

  • Only send messages between pools.
  • Do NOT "send" information to the next step in the same process. It is implied that all necessary information is passed along.
  • Data Object: Piece of data used in a process but goes away once process completes. Linked via an association line.
  • Data Store: Data stored in a database for long-term storage. Association line OUT of data store = query. Association line IN TO data store = update.
  • External pools are usually left blank (we don't have details; DMSU = Don't Model Someone else's Unclear process) — focus on what we can change.

1.11 Common BPMN Mistakes to Avoid (Exam Traps!) REPEATED

Likely on Quiz
  1. Gateways performing actions
  2. Waiting forever (no timer on event gateways)
  3. Using verbs like "Start", "Attempt", etc. in labels
  4. Sending messages within a pool
  5. Unconnected model elements
  6. Jumping process pools
  7. Loop symbol vs. process loop
  8. Duplicating process paths
  9. Two process paths exiting a task (without a gateway)
Class 2 — Lean Six Sigma Process Improvement

2.1 Lean — Core Concept

Lean (core idea, verbatim): Maximize customer value while minimizing waste. (Creating more value for customers with fewer resources.)
Repeated "Organization understands customer value and focuses its key processes to continuously increase it."
"Ultimate goal: provide perfect value to the customer through a perfect value creation process that has zero waste."

2.2 Waste

Waste (verbatim): "Anything other than the minimum amount of equipment, materials, parts, space, and worker's time which are absolutely essential to add value to the product."
The Test "If your customer isn't willing to pay extra for it (or choose your product over another) then it is waste."

Why Focus on Waste?

  • Waste signals that there is time trapped in the process.
  • Recognizing waste leads to identifying the root cause of problems.
  • Underlying any waste is a systemic flaw in the process involving people, materials, machines, and/or information.

Key Stat Typically, 95% of all lead time is non-value-added, with only 5% being value-added.

2.4 The 5 Principles of Lean MEMORIZE

  1. Identify value
  2. Map the value stream
  3. Create flow
  4. Establish pull
  5. Seek perfection

Principle 1: Identify Value (Customers Define Value)

  • Value-added: Any activity that increases the market form or function of the product/service. Activities the customer is willing to pay for.
  • Non value-added: Any activity that does not add market form or function, or is not necessary. Should be eliminated, simplified, reduced, or integrated.

Principle 2: Map the Value Stream

Value Stream: All the activities and resources required to deliver a product/service to the customer — both value-added AND non-value-added activities.

Taking a Value Stream perspective means looking at the big picture, not just individual processes.

Root Cause Analysis ("5 Whys")

Stop when you reach the root cause — the actual root cause will always be the lowest "why" you answered.

Principle 3: Create Flow

Before: process with starts, stops, and delays. After: process with flow (smooth, continuous).

Principle 4: Establish Pull

Push System (verbatim): A push system produces product, using forecasts or schedules, without regard for what is needed by the next operation.
Pull System (verbatim): A method of controlling the flow of resources based on exactly what is needed by the next operation.

Why Use Pull?

  • Flexible and simple method of balancing flow of resources.
  • Effective way to communicate customer demand throughout an operation.
  • More responsive to quality issues.

Takt Time CALCULATION

Takt Time = Work Time Available ÷ Number of Units Sold
Takt Time = Demand Rate

Example: 1800 seconds ÷ 50 coffees = 36.0 sec/coffee

Lean Time Measures

  • Cycle Time (C/T): Time it takes an activity within a service to complete before repeating.
  • Lead Time (L/T): Total time it takes a service to complete from start to finish (includes all activities).
  • Value Added Time (VA): Time spent adding value to a service that the customer is willing to pay for.

Principle 5: Seek Perfection

Seek Perfection = Continuous Improvement. Continuous improvement must be an organization-wide philosophy.

2.5 Six Sigma — Core Concept

Six Sigma (verbatim): Six Sigma is a disciplined, data-driven approach & methodology for eliminating defects. Six Sigma seeks to improve the quality of process outputs by identifying & removing the causes of defects (errors) by minimizing variability in business processes.
Contrast Lean focuses on reducing WASTE. Six Sigma focuses on reducing VARIABILITY.

2.6 DMAIC Framework MEMORIZE

"DMAIC is a methodology for 'filtering out' the trivial many and identifying Critical Factors."
PhasePurposeKey Rule
DefineDefine scope, goals, performance targets. Confirm problem actually exists, is important to customers, is important to the business. Assemble project team.—
MeasureDetermine the "As-Is Condition." Tools: Process map · Forced ranking · Pareto & Cause-and-Effect (Fishbone) diagrams · FMEA (Failure Mode & Effects Analysis).Rule #1: "No Active Manipulation." Data collection is passive; process is studied in its natural state.
AnalyzeProve or disprove what was learned in Measure. Applies statistical logic.Rule #2: "Still No Active Manipulation."
ImproveTeam actively manipulates the remaining X's to gain significant improvement. Validate all results."Start Active Manipulation."
ControlSustain the gains. Training · Procedures · Control Plans · Buy-In · Hand-Off · Audits. Prevent backsliding.Reassemble to revisit; watch for deterioration and for continuous improvement.

DMAIC — Important "X's" Points

  • The whole purpose of DMAIC is to deliver the most useful X's to the Improve Phase.
  • Measure and Analyze collect X's and filter out the less important ones.
  • If all X's were delivered to Improve, it would be overwhelming — so continue to filter.
  • Trap: A really important X may have been discarded. If gains aren't being made in Improve, an important X may have been discarded or never discovered.

2.7 Voice of the Customer (VOC) MEMORIZE

VOC (verbatim):
  • VOC communicates the wants and needs of customers.
  • Must be clearly defined and measurable.
  • Expressed as a target (goal) or upper and lower limits (specifications).
  • A "customer" can be internal or external (another operation, the community, a buying organization, etc.).

2.8 Voice of the Process (VOP) MEMORIZE

VOP (verbatim):
  • A process produces outputs (things), information, and consequences (impact) — collectively called "process outcomes."
  • The "Voice of the Process" refers to measures of process outcomes (measures of results).
  • VOP data must be analyzed to uncover cause & effect relationships.

2.9 The Goal of Six Sigma MEMORIZE

Star Definition "The goal of Six Sigma is to align the voice of the customer with the voice of the process 99.9997% of the time — no more than 3.4 defective parts per million opportunities (DPMO)."

2.10 The 3-Sigma Math Problem

30 components, each with 3-sigma quality (99.75%), all must work:
P(all work) = 0.997530 ≈ 0.9277 = 92.77%
Defect rate = 1 − 0.9277 ≈ 7.23%

Insight: Even tiny component defect rates compound across complex products — which is why Six Sigma (99.99966%) is needed.

2.11 Lean Six Sigma — Used Together

Trap "Lean and Six Sigma should be used in tandem, NOT combined." Managers need to balance Lean and Six Sigma teams to promote collaboration. Complementary & synergistic goals.
Class 3 — Digital BPM, RPA, AI, GANs & RAG

3.1 Business Process Management (BPM)

BPM (Definition 1, verbatim): Focuses on improving corporate performance by managing and optimizing a company's business processes. Can be described as a "process optimization process." BPM enables organizations to be more efficient, more effective, and more capable of change than a functionally-focused, traditional hierarchical management approach.
BPM (Definition 2, verbatim): BPM is a discipline that uses various methods to discover, model, analyze, measure, improve and optimize business processes. A business process coordinates the behavior of people, systems, information, and things to produce business outcomes in support of a business strategy.

As a managerial approach, BPM sees processes as strategic assets of an organization that must be understood, managed, and improved to deliver value-added products and services to clients.

BPM Benefits

  • Increased efficiency
  • Reduced costs
  • Improved customer satisfaction
  • Better regulatory compliance
  • Faster process execution
  • Improved data accuracy
  • Greater visibility into operations
  • Ability to streamline & optimize business processes

3.2 Digital Business Process Management

Digital BPM (verbatim): Converting manual processes into digital processes and managing them using digital technology.

Digital BPM enables two key outcomes:

  • Process Execution — BPMN leads to process execution.
  • Process Intelligence — BPMN leads to process intelligence (including process mining).

3.3 The 7 Rs of Process Innovation (Steven Shapiro) MEMORIZE

  1. Rethink
  2. Reconfigure
  3. Resequence
  4. Relocate
  5. Reduce
  6. Reassign
  7. Retool

3.4 Robotic Process Automation (RPA)

RPA (verbatim): Use call activities or bots to perform automated tasks or series of tasks.
  • RPA usually employs preprogrammed bots that follow specific rules.
  • In business process modeling, we call pre-specified rules "rules engines".
  • Rules engines are NOT Artificial Intelligence.
  • RPA may employ basic AI functions (summarization, automated response generation, etc.).
Trap "RPA does NOT delegate critical thinking or decision making to bots." That's where Intelligent Automation / Agentic AI begins.
Repeated "Automating Tasks DOES NOT Change the Process." (Said 4+ times across Class 3 and Class 4.)

3.5 Intelligent Automation — The Bridge

Intelligent Automation (verbatim): RPA with decision-making AI.

3.6 Artificial Intelligence (AI)

AI (verbatim, core definition): Machine behavior & function that exhibits the intelligence & behavior of humans.

How AI Works (7 Steps)

  1. Data Collection — Requires extensive data (images, text, numbers) for learning.
  2. Data Processing — Cleans and organizes data for better understanding.
  3. Algorithms — Step-by-step instructions for data analysis.
  4. Model Training — Learns from data to refine predictions.
  5. Feedback and Adjustment — Improves through error feedback.
  6. Pattern Recognition — Identifies patterns for predictions.
  7. Deployment — Implemented in real-world scenarios, continuing to learn from new data.

3.8 Generative AI

Generative AI (verbatim): Trainable algorithms that create (generate) new content, including audio, code, images, text, simulations, and videos.

Generative AI learns from existing artifacts to generate new, realistic artifacts (at scale) that reflect the characteristics of the training data but don't repeat it.

Content types: Images · Videos · Music · Speech · Text · Software code · Product designs.

GenAI Inflection Point

  • ELIZA chatbot — Created 1961. Simulated a psychotherapist using natural language.
  • Generative AI Inflection Point = 2014 — Introduction of Generative Adversarial Network (GAN).

3.9 Generative Adversarial Network (GAN) MEMORIZE

GAN (verbatim):
  • Generative AI creates content.
  • Generative AI tries to identify it as GenAI-created.
  • Feedback loop creates convincing content.

Think of it as one AI generating, another AI critiquing — they compete, and the generator gets better at fooling the discriminator over time.

3.10 Retrieval-Augmented Generation (RAG) MEMORIZE

RAG (verbatim): LLM references additional data from a dataset beyond the original (common/shared) training data.

Why important: RAG allows a general-purpose LLM to answer questions about proprietary or recent information it wasn't trained on — critical for business applications of AI.

4.3 Agentic AI MEMORIZE

Agentic AI (core definition, verbatim): An AI system specifically designed to understand complex workflows and achieve goals autonomously, with little to no human interference.

5 Key Characteristics of Agentic AI

CharacteristicDefinition (verbatim)
AutonomyImbued with its own goals, objectives, and motivations that drive decision-making and actions. Autonomously chooses most appropriate steps to pursue goals without human supervision.
Adaptive Learning and ReasoningGathers information from interactions with environment and learns from past experiences. Applies reasoning and problem-solving to update knowledge, revise strategies, and improve performance over time.
Language UnderstandingLeverages state-of-the-art NLP models to process and comprehend human language at a deep semantic level.
Workflow OptimizationCombines language understanding with reasoning, planning, and decision-making abilities to optimize workflows and business processes (e.g., optimizing resource allocation, streamlining communication and collaboration, identifying opportunities for process automation).
Multi-agent and System ConversationCommunicates between different agents to build a complex workflow. Connects to other systems or tools (email, code executor, search engine) to perform variety of tasks.

Common AI Agents

  • Code generators
  • Customer service prioritization
  • Social media / blog post generator
  • News article writer
  • Spam detector
  • Job description generator

More Advanced AI Agents

  • Automated stock trading

AI Agent Example — "John's Birthday Agent"

John's AI agent: recognizes upcoming birthday → determines gift giving is appropriate → scrapes digital correspondence & social media to identify father's preferences → identifies best value-for-money gift → orders gift for delivery and writes heartfelt note in John's "voice" → reminds John about the birthday.

Repeated 4+ times "Automating Tasks DOES NOT Change the Process." Agentic AI enables actual process change — not just task automation.

4.4 Machine Reengineering MEMORIZE

Machine Reengineering — 2 Concepts (verbatim):
  1. Redesigning business processes so humans work with AI agents to improve effectiveness and/or efficiency.
  2. AI acts as business process engineer to optimize business processes.
"Managers noted historical trends and revised processes, and engineers developed code that was then baked into computing systems. Every update or response to the market required multiple steps; it cost time and performance. Sometimes, by the time changes were in place, the market had already moved."

"Machine-reengineering asks that people train and actively manage the performance of the algorithms and data models that drive process change, rather than drive process change themselves."
— Wilson, Alter, Shukla et al. (2016)
Class 5 — Service Thinking & Value-Based Innovation

5.1 Service Thinking — Core

Service Thinking (verbatim): The purpose of a service is to help customers solve a problem and help them to achieve certain desired outcomes or goals. Customers have problems that they need solved.

3 Questions of Service Thinking MEMORIZE

  1. What is the problem to be solved?
  2. What is the service interface?
  3. What are proper measures of service quality?

5.2 Outside-In Thinking MEMORIZE

Outside-In Thinking (verbatim, appears in both Class 5 & Class 6): Outside-In thinking focuses on solving customer problems instead of focusing on internal operations.

The Outside-In Perspective Constantly Asks:

  • What do customers need?
  • How and why are customers changing?
  • What can we do to better solve their problems & meet their emerging needs?
Repeated from Class 1 "Who are your customers? You don't get paid for what you do. You get paid for the value you create."

5.3 PTBS (Reminder)

PTBS (verbatim, same as Class 1): The elemental / core reason the service is valuable to the consumer of the service. It is often not an obvious or surface-level reason.

PTBS ~ Job to be done (equivalent concepts).

5.4 Service Interface

Service Interface (verbatim): Customers interact with a service via the service interface. A service interface consists of all the interactions between the customer and the service process engine (which includes all partner service process engines).

5.5 Service Metrics — Two Kinds

  • Value metrics — measure value delivered to the customer (at the top).
  • Process metrics — measure internal efficiency.
Process Productivity (verbatim): A service is productive to the extent that the service process engine uses the least amount of resources in the least amount of time to satisfy the customers' desired outcomes.

The objective is to minimize the cost of satisfying the customers' desired outcomes, thereby maximizing service profitability.
Trap "Don't stare in the mirror." Don't focus on process metrics at the expense of value metrics. (Reappears as one of the 7 Principles in Class 6.)

5.6 Service-Dominated Logic (SDL)

Goods-Dominated LogicService-Dominated Logic
Separation between producer & consumer. Maximizes production control, efficiency, and profit maximization. Achieved by standardizing product and producing it away from the market.Focused on the interaction of the producer and the consumer and other supply/value network partners as they co-create value through collaborative processes. Driven by innate purpose of doing something for and with another party — customer-centric and customer-responsive.

Tangibles vs. Intangibles

  • A tangible good enables customers to provision a service that can satisfy the customers' desired outcomes with respect to a job-to-be-done.
  • Experience is fundamentally about the intangible, not the tangible.
  • Shift from stand-alone product to solution for customer.

Key SDL Principle

"Goods are actually mechanisms for service provision. The customer is always buying a service flow rather than a tangible thing; thus the firm should focus on selling a flow of service. Organization should focus on the whole service system rather than a stand-alone product."

  2. Increase Reach & Engagement

Remember "Everyone Will Know About Your Failures." Customer experiences are shared widely in a digital world.

5.8 Value-Based Innovation MEMORIZE

Value-Based Innovation (verbatim):
  • Creating a more compelling value proposition.
  • Finding a better way to solve customer's PTBS.

Value-based innovation ≠ invention. Value-based innovation focuses on delivering a better solution to the customer's problem — the tech may already exist.

Warning "Companies are at risk when they start focusing on what they do — rather than delivering maximum value to their customers."

Value Creation Transcends Industry Borders

  • Craigslist destroyed classified advertising in newspapers.
  • Skype + WhatsApp sucked $386 billion away from telcos.
  • When you serve the customer's PTBS better, you steal value from adjacent industries.

5.9 Combinatorial Innovation MEMORIZE

Combinatorial Innovation: Combining existing ideas, technologies, or components in new ways to create value.

Examples: Toyota Prius (hybrid patented in 1889 + modern electronics, introduced 1997). Smartphone navigation = GPS + smartphone hardware + map software.

Key Stat "Only 10% of business model innovations introduced new business model patterns. 90% of business model innovations merely adapted, refined, or combined previous business model patterns." — Univ. of St. Gallen

6.2 The 5 Digital Forces CORE MEMORIZE

Gimpel & Westerman (2012); Cognizant & MIT (2012).

1. Globalization

Globalization (verbatim): Globalization is more than supply chain management and optimizing labor costs. It fundamentally changes the nature of supply and demand.

As artificial barriers disappear, consumers enjoy a new world of choice no longer limited by geographic borders. Companies benefit from larger markets and global sourcing, while concurrently taking steps to fend off challenges from foreign competitors.

2. Millennialization

Customers no longer trust corporate propaganda — they ask friends, strangers, and social networks. Millennialization facilitates an outside-in perspective.

  • "Here's what I'll tell you about our product" → "I don't trust your propaganda" → "I'll ask my friends. Heck, I'll ask strangers."

3. Prosumerization

Prosumerization (verbatim): Inexpensive, easy-to-use tools provide professional-grade results, lower barriers to market entry. Prosumerization changes the nature of supply and demand.

4. Virtualization

Virtualization (verbatim): Business virtualization empowers firms to focus on their core strengths by partnering with other firms that can better execute other business processes.

When pursued mindfully, virtualization can reduce costs, increase agility, and boost quality by moving some process steps to companies that excel in those specific functions.

Examples: Outsourcing & temporary alliances.

5. Platformization

Platformization (core quote): "Today platforms are the gateways through which businesses must pass to reach their customers."
Platform (verbatim definition): An open architecture with a governance model designed to facilitate interactions.

6.3 Network Effects MEMORIZE

Network Effect (verbatim): The value of a network depends on the number of users with whom members can interact.

Network effects can be both positive AND negative. The number and type of users makes a difference.

6.5 Characteristics of Industries Ready for Platformization

  • High proportion of value from information
  • Highly fragmented industries
  • Simple, modular products & services (connectable through APIs, etc.)
  • Existing untapped spare capacity in market

Platforms are often layered on top of other platforms. Platforms in one layer are often a critical chokepoint for platforms in another layer.

6.6 Winner-Takes-All MEMORIZE TRAP

Trap — All 4 REQUIRED Winner-Takes-All Requires (ALL 4 MUST be present):

  2. Strong network effects
  3. Low demand for differentiated features
  4. High multi-homing costs
Counterintuitive Trap "First Mover Advantage is a MYTH." First-to-market does not guarantee victory, even when "first to market wins" is commonly repeated. Expect a question on this.

⚠️ Master Exam-Trap List — Read Before the Quiz

These are items Gimpel explicitly flagged, repeated, or structured as common mistakes — high-probability quiz hits.

BPMN Traps

  1. Gateways don't make decisions or perform actions — they route traffic. The data/task before the gateway made the decision.
  2. Throwing = filled, Catching = not filled. Counterintuitive.
  3. Events "just happen"; tasks are assigned to people or machines.
  4. Start = thin border; Intermediate = double border; End = thick border.
  5. Event-based gateway is the ONLY future-looking gateway. Always put a timer event to avoid waiting forever. Exiting paths recombined with XOR.
  6. Interrupting boundary event → ONLY exception flow (current process stops). Non-interrupting → both flows continue.
  7. Messages go ONLY between pools. Never within a pool.
  8. Processes cannot jump pools. One pool = one process (split by lanes).
  9. Activity labels = imperative verbs, not descriptions. Do not write who performs the task — the swim lane shows that.
  10. Avoid "Start", "Attempt" in activity labels. Listed on Common Mistakes slide.
  11. Parallel gateways don't need labels — all paths are taken, so logic is clear.
  12. Don't use AND gateway to join into a none-end event — join is already implied.
  13. Only send messages between pools (never within); don't send data to next step in same process — information is implied.
  14. Rules engines are NOT AI. Preprogrammed rules.

Lean Six Sigma Traps

  1. Waste definition must be verbatim. "Anything other than the minimum amount of equipment, materials, parts, space, and worker's time which are absolutely essential to add value to the product."
  2. The customer-pay test is the ultimate waste test: if customer won't pay extra (or choose your product), it's waste.
  3. Lean = waste. Six Sigma = variability. Different targets.
  4. Lean and Six Sigma are used IN TANDEM, NOT COMBINED.
  5. DOWNTIME — all 8 letters in order. Know "Employees Underutilized" isn't "Excess."
  6. 95% of lead time is non-value-added. Numeric fact.
  7. DMAIC: Measure & Analyze = NO active manipulation. Improve = START active manipulation.
  8. VOC can be internal or external. (Another operation, community, buying org.)
  9. 99.9997% = 3.4 DPMO. Both numbers memorized.
  10. 3-sigma math: 30 components @ 99.75% → 92.77% work, 7.23% defective.
  11. Takt Time formula: time available ÷ units demanded. DO NOT confuse with cycle time.
  12. Cycle vs. Lead vs. Value-Added Time are distinct — know all three.
  13. DMAIC "filters X's." Important X may have been discarded — use this if Improve shows no gains.

AI / RPA / Agentic Traps

  1. RPA ≠ AI. RPA uses rules engines.
  2. "Automating tasks DOES NOT change the process." Said 4+ times.
  3. Intelligent Automation = RPA + decision-making AI.
  4. 7 AI Patterns are NOT MECE. Overlaps exist — don't say they're mutually exclusive.
  5. GenAI inflection point = 2014 (GAN invention).
  6. GenAI creates new artifacts that DON'T repeat the training data but reflect its characteristics.
  7. RAG = LLM references additional data beyond original training data.
  8. Agentic AI keyword: "autonomously" / "little to no human intervention."
  9. Machine Reengineering has TWO concepts, not one.

Service / Digital Traps

  1. Outside-In ≠ customer satisfaction surveys. It's a constant ask: what's changing, what do customers need, how can we solve better.
  2. 4 Customer Experience Transformation aspects: Outside-In Design · Increase Reach & Engagement · Data-driven Experience · Seamless Physical-Digital Integration. Memorize.
  3. Value-based innovation ≠ invention.
  4. Combinatorial Innovation: 90% of business model innovations merely combined/adapted previous patterns. Only 10% were net-new.
  5. 5 Digital Forces — spell correctly: Platformization, Globalization, Millennialization, Prosumerization, Virtualization.
  6. Platform def must be verbatim: "An open architecture with a governance model designed to facilitate interactions."
  7. Winner-Takes-All needs ALL 4 conditions. 3 aren't enough.
  8. First-Mover Advantage is a MYTH. Huge trap — everyone's intuition says first wins.
  9. Startup + advertising = bad idea (two reasons).
  10. 2 Critical Success Factors = Digital Capabilities + Leadership. Not three.
  11. Change operational processes FIRST. Not customer experience first — Gimpel repeats: customers can SEE customer-experience changes but NOT operational changes, so operational changes are your moat.
  12. Leadership MUST be top-down.
  13. Vision must be transformative, not incremental.
  14. 3 Types of Digital Vision: customer experience, operational processes, business model.

🧠 Practice Quiz — 30 Questions Across All 6 Classes

Click "Show Answer" under each question to check. Answers reference slides verbatim.

BPMN (Class 1)

Q1. Why is a "Send Task" symbol filled in / darkened?
Because it is a throwing (sending) symbol. Throwing symbols are filled. Receiving (catching) symbols are NOT filled.
Q2. Can a gateway decide which path to take based on new information it computes?
No. Gateways do not make decisions or allow time to pass. They direct traffic. The preceding activity/data must have determined the answer. Gateway logic is "pure logic; result based on data previously obtained/analyzed."
Q3. What is the only "future-looking" gateway?
The Event-based gateway. "Events are in the future! It's the only future-looking gateway." Every path out has an event on it; gateway logic = whichever event happens first.
Q4. What's the difference between a Standard Loop and a Multi-Instance Loop?
Standard Loop: same activity repeated until a condition becomes true; one instance at a time.
Multi-Instance Loop: multiple instances of the same activity run (parallel or sequential), once for each object in a collection.
Q5. What's the difference between an interrupting and non-interrupting boundary event?
Interrupting: stop the current process, follow ONLY the exception flow.
Non-interrupting: keep the current process going AND ALSO follow the exception flow.
Q6. Can a process use a message flow to "cross" between two swim lanes in the same pool?
No. "Do not send messages within a pool!" Messages flow ONLY between pools. Within a pool, information is implied to be passed along.
Q7. Which BPMN task is performed by a human WITHOUT the aid of any business process execution engine or application?
Manual Task. (A "User Task" is performed by a human with software assistance.)

Lean Six Sigma (Class 2)

Q8. State the verbatim definition of waste.
"Anything other than the minimum amount of equipment, materials, parts, space, and worker's time which are absolutely essential to add value to the product."
Q9. List the 8 DOWNTIME waste categories.
Defects · Overproduction of work in process · Waiting · Non-value-added processing · Transportation of materials & equipment · Inventory excesses · Motion · Employees underutilized.
Q10. A coffee shop is open 30 minutes and sells 50 coffees. What is takt time?
Takt Time = Work Time Available ÷ Number of Units Sold = 1,800 seconds ÷ 50 = 36.0 seconds per coffee.
Q11. In DMAIC, which phases forbid active manipulation?
Measure (Rule #1: "No Active Manipulation") and Analyze (Rule #2: "Still No Active Manipulation"). Active manipulation begins in the Improve phase.
Q12. What is the numerical goal of Six Sigma?
Align the voice of the customer with the voice of the process 99.9997% of the time — no more than 3.4 DPMO (defective parts per million opportunities).
Q13. Push vs. Pull — define each.
Push: Produces product using forecasts or schedules, without regard for what is needed by the next operation.
Pull: A method of controlling the flow of resources based on exactly what is needed by the next operation.
Q14. If a product has 30 components each at 3-sigma (99.75%) quality, what percent of final products work?
(0.9975)30 ≈ 0.9277 → 92.77% work properly. About 7.23% are defective. This is why Six Sigma matters for complex products.
Q15. Are Lean and Six Sigma "combined" into Lean Six Sigma?
No. They should be used in tandem, NOT combined. Managers balance Lean and Six Sigma teams to promote collaboration.

AI / BPM / RPA (Classes 3 & 4)

Q16. What are the 7 Rs of Process Innovation (Shapiro)?
Rethink · Reconfigure · Resequence · Relocate · Reduce · Reassign · Retool.
Q17. Define RPA.
RPA = Robotic Process Automation: use call activities or bots to perform automated tasks or series of tasks. Uses preprogrammed rules engines (NOT AI). Does NOT delegate critical thinking or decision making.
Q18. What is Intelligent Automation?
Intelligent Automation = RPA with decision-making AI.
Q19. Explain how a GAN works.
Generative AI creates content. Another Generative AI tries to identify it as GenAI-created. A feedback loop between the two creates convincing content. (The introduction of GANs in 2014 was the generative-AI inflection point.)
Q20. Why is RAG important for business applications of AI?
RAG (Retrieval-Augmented Generation) lets an LLM reference additional data from a dataset beyond the original training data. That's how a general-purpose model can answer questions about your proprietary or recent data — essential for enterprise use.
Q21. List the 5 key characteristics of Agentic AI.
(1) Autonomy, (2) Adaptive Learning and Reasoning, (3) Language Understanding, (4) Workflow Optimization, (5) Multi-agent & System Conversation.
Q22. What are the two concepts behind Machine Reengineering?
(1) Redesigning business processes so humans work with AI agents to improve effectiveness and/or efficiency. (2) AI acts as the business process engineer to optimize business processes.

Service Thinking / Value-Based Innovation (Class 5)

Q23. Define Outside-In Thinking.
Outside-In thinking focuses on solving customer problems instead of focusing on internal operations. It constantly asks: What do customers need? How and why are customers changing? What can we do to better solve their problems & meet their emerging needs?
Q24. What are the 4 required aspects of successful Customer Experience Transformation?
(1) Outside-In Design, (2) Increase Reach & Engagement, (3) Data-driven Experience, (4) Seamless Physical-Digital Integration.
Q25. Define Value-Based Innovation and contrast it with invention.
Value-Based Innovation = creating a more compelling value proposition + finding a better way to solve the customer's PTBS. It is NOT invention — technologies and ideas may already exist; the innovation is in how they deliver more value. 90% of business model innovations are combinations of existing patterns.

Digital Transformation (Class 6)

Q26. Name and briefly describe the 5 Digital Forces.
Globalization (fundamentally changes supply & demand across geographies); Millennialization (customers distrust propaganda; ask friends/strangers; facilitates outside-in perspective); Prosumerization (inexpensive pro-grade tools lower market entry barriers); Virtualization (firms focus on core strengths, partner with others to execute non-core processes); Platformization (platforms are the gateway through which businesses reach customers).
Q27. What 4 conditions must ALL be present for a Winner-Takes-All outcome?
All 4 required: (1) High economies of scale, (2) Strong network effects, (3) Low demand for differentiated features, (4) High multi-homing costs.
Q28. True or false: First-Mover Advantage guarantees winning in platform markets.
FALSE. "First Mover Advantage is a Myth." This is explicitly called out as counterintuitive in the slides.
Q29. List the 5 Business Model Reinvention Archetypes.
(1) Reinvent Industries, (2) Substitute Products or Services, (3) Create New Digital Businesses, (4) Reconfigure Value Delivery, (5) Rethink Value Propositions.
Q30. Why does Gimpel say to change operational processes FIRST in digital transformation?
Operational-process change is (a) the foundation for future changes, and (b) hard to duplicate. Competitors can see how you transform the customer experience, but they cannot see how you transform your operations — so operational changes are your moat.
